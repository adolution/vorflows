Drei Module · Ein System

Base baut. SEOFlow rankt. ClarityFlow optimiert.

Base ist das Fundament. SEOFlow und ClarityFlow sind optionale Add-Ons.

01 / Base · Pflicht-Modul

Theme umbauen, ohne dass dein Live-Shop wackelt.

Duplicate Theme, Claude Project laden, Sections und Snippets bearbeiten.

Änderungen landen in einem Duplicate Theme. Erst nach Review wird published. Skill-Stack inklusive.

02 / SEOFlow · Add-On

SEO-Maschine, die in 10 Minuten steht.

Geführter Setup-Flow. Reale Suchdaten mit der SEO-KI verknüpft.

Echte Suchdaten, Rankings, CTRs als Input. Output: Handlungsempfehlungen oder direkte Code-Änderungen am Theme.

03 / ClarityFlow · Add-On

Dein Shop verrät, was nicht funktioniert. Claude hört zu und handelt.

Verbindet Claude Code mit Verhaltensdaten: Klicks, Scroll, Heatmaps.

Erkennt Drop-offs, ignorierte CTAs, tote Sections. Schließt den Loop: Daten → Hypothese → Code → neue Daten.

Was im Paket liegt

Drei Dateien. Ein System, das mit dir mitwächst.

Wochenlang gebaut, getestet, geschärft. Du gibst einen Befehl, die KI führt dich durchs Setup.

setup.md

Auto-Installer

Ein Befehl. Skills, Plugins, Konfig. Alles landet in deinem Claude-Workspace, fertig konfiguriert.

# einmal ausführen
$ claude setup.md
# → Skills + Plugins ready
CLAUDE.md

Mein Playbook für deine KI

Wochenlang an echten Shop-Cases geschärft. Sagt deiner KI, wie sie arbeiten soll: welche Patterns funktionieren, welche nicht, welche Reihenfolge sicher ist.

# Anweisungen
always: Backup anlegen
never: Code ohne Plan schreiben
prefer: editable section
memory.md

Lernt dich kennen

Speichert Vorlieben, Constraints, Shop-Eigenheiten. Aktualisiert sich automatisch nach jedem Chat. Claude vergisst nichts.

# nach 3 Sessions
brand_color: "#C8633E"
tone: "direkt, knapp"
avoid: "edits in file xy"
Sicher by design. Claude arbeitet immer auf einem Duplikat deines Themes, niemals direkt am Live-Shop. Kein Zugriff auf Kundendaten, keine API-Keys. Du kopierst Dateien gelegentlich rüber und entscheidest selbst, wann published wird.

FAQ

Was du dich zurecht fragst.

Brauche ich Programmier-Erfahrung?
Nein. Wir haben vorflows mit Foundern ohne Coding-Background getestet. Alle haben am Ende exakt den Output bekommen, den sie haben wollten.
Was, wenn ich mein Theme zerschieße?
Du arbeitest immer auf einem Duplicate. Live-Theme bleibt unangetastet, bis du publishst. Kein Auto-Deploy.
Funktioniert das mit jedem Theme: Dawn, Custom, Premium?
Ja. Solange es ein Shopify-Theme ist. Die Setup-Datei von vorflows liest dein komplettes Theme, kategorisiert es und merkt sich, wie es funktioniert.
Brauche ich Claude Pro / Claude Code?
Ja, einen Claude-Account mit Code-Zugang. Du bekommst ein kurzes 5-Minuten-Video, in dem du das Setup nachklicken kannst.
Kann ich später Add-Ons dazukaufen?
Ja. Aber: Add-Ons nach Erstkauf gibt's ohne Bundle-Rabatt. Wer alles will, fährt mit dem Bundle besser.
Wie lange dauert das Onboarding?
Base: 20 bis 30 Minuten Setup, erste Section am selben Tag. ClarityFlow: 15 Minuten Setup. SEOFlow: 30 bis 40 Minuten Setup.
Was passiert mit meinen Daten?
SEO- und Verhaltensdaten bleiben in deinen Accounts. vorflows hostet keine Shop-Daten. Alles läuft über deine eigenen Verbindungen zu Shopify, SEO-Tools und Claude. Wenn du dich unwohl dabei fühlst, dass Claude deine Daten verarbeitet, ist vorflows nicht das richtige für dich.
Gibt es eine Geld-zurück-Garantie?
Das ist zwar bisher nie passiert, aber: Wenn du das Setup nach Anleitung durchführst und das System nicht für dich funktioniert, bekommst du dein Geld zurück.
