Updates Was passiert bei einem Theme-Update?

Ein Theme-Update überschreibt Theme-Dateien, deine individuellen Code-Anpassungen können dabei verloren gehen. So übernimmst du sie sauber ins neue Theme:

  1. Sicherung anlegen. In Shopify unter Onlineshop → Themes dein aktuelles Theme (mit deinen Anpassungen) duplizieren. Damit hast du jederzeit einen Rückfallpunkt.
  2. Update einspielen. Spiel die neue Theme-Version ein. Jetzt hast du zwei Stände: dein altes Theme mit den Anpassungen und das neue, aktualisierte.
  3. Claude vergleichen lassen. Öffne beide Stände in Claude Code und bitte Claude, sie zu vergleichen und deine Anpassungen ins neue Theme zu übertragen.
    Beispiel-Prompt

    „Ich habe mein Theme aktualisiert. Vergleiche die alte Version (mit meinen Anpassungen) mit der neuen Version und übertrage alle meine individuellen Code-Änderungen sauber ins neue Theme. Liste mir vorher auf, was du überträgst."

  4. Testen, dann veröffentlichen. Vorschau prüfen (Produktseite, Warenkorb, Startseite). Passt alles, das neue Theme veröffentlichen.

Nie das Live-Theme direkt updaten. Immer am Duplikat arbeiten und erst nach dem Test veröffentlichen, so ist dein Shop nie kaputt.

Updates Bekomme ich kommende Updates?

Ja. Updates bekommst du per E-Mail mit einem Download-Link. Du entpackst die Datei, ziehst die enthaltene Update-Datei in Claude Code und bittest Claude, das Update einzuspielen.

Deine eigenen Einstellungen, dein Theme und deine Daten bleiben dabei erhalten, es wird nichts überschrieben, nur ergänzt.

Einrichtung Sollte das Projekt in OneDrive oder einer Cloud liegen?

Besser nicht. Cloud-Ordner wie OneDrive synchronisieren ständig und sperren Dateien kurzzeitig, während Claude arbeitet. Das kann zu Lese- oder Cache-Fehlern führen.

Leg das Projekt in einen normalen lokalen Ordner (z. B. unter „Dokumente"). Sichern darfst du es natürlich trotzdem, nur nicht direkt im aktiven Arbeitsordner synchronisieren lassen.
