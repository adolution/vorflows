Hilfe bei Fehlern

Lösungen für die häufigsten Fehlermeldungen. Findest du dein Problem hier nicht, schreib mir kurz.

Installation „claude is not recognized" oder „process exited with code 1"

Beide Meldungen haben dieselbe Ursache: Claude Code ist noch nicht fertig eingerichtet. Entweder ist der Befehl claude nicht installiert, oder die aktualisierten Nutzungsbedingungen wurden noch nicht bestätigt. Das passiert genau einmal und ist in wenigen Minuten erledigt.

  1. Terminal öffnen. In VS Code über das Menü Terminal → Neues Terminal (oder am PC: Windows-Taste drücken, powershell tippen, anklicken).
  2. Claude installieren. Kopier die Zeile für dein Gerät, ins Terminal einfügen, Enter.
    Windowsirm https://claude.ai/install.ps1 | iex
    Maccurl -fsSL https://claude.ai/install.sh | bash
  3. Neues Terminal. Das alte Terminal schließen und neu öffnen. Erst dann wird claude erkannt.
  4. Einrichtung komplett durchklicken. Tipp claude, Enter. Die kurze Einrichtung ganz zu Ende machen, nicht abbrechen: ein Design mit den Pfeiltasten wählen (z. B. „Dark mode") und Enter, dann im Browser anmelden, danach die Nutzungsbedingungen mit Enter bestätigen. Warte, bis unten die normale Eingabezeile erscheint — erst dann das Terminal schließen. Danach VS Code einmal neu starten, der Fehler ist weg. (Mehr dazu im letzten Punkt unten.)

Wichtig: Im Browser-Login oder in der Claude-Desktop-App zu bestätigen reicht nicht. Die Bestätigung muss in Claude Code selbst passieren (Schritt 4).

Installation „claude" wird nach der Installation nicht erkannt

Das Terminal kennt den neuen Befehl erst nach einem Neustart. Schließ das aktuelle Terminal komplett und öffne ein neues über Terminal → Neues Terminal. Tipp claude erneut ein.

Hilft das nicht, schließ VS Code einmal ganz und öffne es neu. Danach wird der Befehl erkannt.

Stand beim Installieren ein gelber Hinweis zu „PATH"? Dann hilft der nächste Punkt.

Windows Windows: „claude" ist installiert, wird aber trotzdem nicht erkannt

Lief die Installation mit „successfully installed" durch, steht darunter aber ein gelber Hinweis wie „… is not in your PATH"? Dann ist Claude korrekt installiert — Windows weiß nur noch nicht, in welchem Ordner das Programm liegt. Deshalb meldet claude „nicht erkannt". So behebst du es:

  1. Diese Zeile einfügen. Kopier sie, klick ins selbe Terminal, füg sie ein, drück Enter. Es kommt keine Rückmeldung — das ist normal und richtig.
    Windows$p=[Environment]::GetEnvironmentVariable("Path","User"); [Environment]::SetEnvironmentVariable("Path", $p + ";$env:USERPROFILE\.local\bin", "User")
  2. Neues Terminal. Das aktuelle Terminal komplett schließen (Mülleimer-Symbol) und ein neues öffnen über Terminal → Neues Terminal.
  3. Claude starten. Tipp claude, Enter → im Browser anmelden → die Nutzungsbedingungen mit Enter bestätigen. Fertig.

Nur schnell testen, ohne den PATH zu ändern? Gib einmalig & "$env:USERPROFILE\.local\bin\claude.exe" ein. Mit den drei Schritten oben läuft claude aber dauerhaft überall.

Einrichtung Rote Meldung „exited with code 1" und „You must run claude in the terminal to review the updated terms"

Oben erscheint der rote Hinweis „Error: Claude Code process exited with code 1", unten rechts dazu „… You must run claude in the terminal to review the updated terms" (oft mit Button „Resolve in Terminal"). Beides hat dieselbe Ursache: die einmalige Claude-Einrichtung im Terminal wurde noch nicht abgeschlossen — meist, weil beim Auswählen des Designs das Terminal zu früh geschlossen wurde. Genau dieser Bildschirm muss komplett durchgeklickt werden:

  1. Einrichtung starten. Klick auf den blauen Button „Resolve in Terminal" in der Meldung. Oder öffne ein Terminal (Terminal → Neues Terminal) und tipp claude, Enter.
  2. Design wählen. Beim Willkommensbildschirm mit den Pfeiltasten ein Design auswählen (z. B. „Dark mode") und Enter.
  3. Anmelden. Falls gefragt: der Browser öffnet sich → mit deinem Konto einloggen → zurück ins Terminal.
  4. Bedingungen bestätigen. Claude zeigt die aktualisierten Nutzungsbedingungen → mit Enter bestätigen.
  5. Fertig abwarten. Warte, bis unten eine normale Eingabezeile zum Tippen erscheint (Claude ist bereit). Erst dann das Terminal schließen und das Plugin wieder benutzen.

Wer mitten in der Einrichtung abbricht oder das Terminal zu früh schließt, bekommt wieder dieselbe Meldung. Im Browser oder in der Desktop-App zu bestätigen zählt hier nicht — es muss in Claude Code im Terminal passieren.

Plugins Plugin lässt sich nicht hinzufügen: „/plugin" wird nicht erkannt

/plugin-Befehle gehören zu Claude Code selbst. Sie funktionieren nur in einer laufenden Claude-Sitzung im Terminal — nicht im leeren Terminal davor und nicht im Chatfenster des Plugins. Tippst du sie ins leere Terminal, kommt „'/plugin' wurde nicht als Cmdlet … erkannt"; im Plugin-Chat sinngemäß „not supported in this environment".

  1. Terminal öffnen. In VS Code: Terminal → Neues Terminal.
  2. Claude starten. Tipp claude, Enter. Warte, bis oben „Claude Code v2.x" steht und unten eine Eingabezeile mit „>" erscheint.
  3. Erst jetzt den Plugin-Befehl eingeben, direkt in diese Claude-Eingabezeile, z. B.:
    /plugin marketplace add AgriciDaniel/claude-seo

Kurz: zuerst claude starten, dann den /plugin-Befehl. Nicht andersherum.

Plugins Plugin-Installation: „Command 'git' not found or is in an unsafe location"

Zum Installieren der Plugins lädt Claude die Daten per Git. Diese Meldung heißt: Git ist auf dem Rechner nicht installiert (oder nicht auffindbar). Git ist kostenlos und in zwei Minuten eingerichtet.

  1. Git installieren. Zeile fürs Gerät ins Terminal einfügen, Enter:
    Windowswinget install --id Git.Git -e --source winget
    Macxcode-select --install
    Windows alternativ: den Installer von git-scm.com laden und alle Vorgaben bestätigen.
  2. Neu starten. Terminal schließen und ein neues öffnen (oder VS Code neu starten), damit git erkannt wird. Prüfen mit git --version — kommt eine Versionsnummer, passt es.
  3. Erneut versuchen. claude starten und den Plugin-Befehl nochmal eingeben (/plugin marketplace add …).

Kommt die Meldung trotz installiertem Git und nennt „unsafe location"? Dann liegt das Projekt in einem Cloud-/Spezialordner (z. B. OneDrive). Verschieb es in einen normalen lokalen Ordner und versuch es erneut.
