Shopify-Stores werden heute auf zwei Wegen gebaut. Entweder über eine Agentur, die zu viel kostet und zu langsam liefert. Oder über einen Stack aus 15 Apps, der jeden Monat 500 € abzieht — und dem Founder am Ende nichts gehört.

vorflows ist der dritte Weg. Ein geführter Workflow, der Claude Code zum technischen Co-Founder des Shops macht. Theme als File-Export, Änderungen über Claude, dupliziertes Theme, Review, Publish. Keine CLI-Magie. Keine Agentur. Kein App-Stack.

Was ein Senior Dev kann, kann der Workflow. Nur ohne Salär, ohne Retainer, ohne 15 Apps, die du nicht besitzt.

// workflow

In drei Schritten vom Theme-Export zum publizierten Build.

Nichts läuft an deinem Live-Theme vorbei. Jede Änderung passiert in einer Kopie, wird reviewt und erst dann published.

01 EXPORT

Theme als File-Export aus Shopify. Roh, vollständig, lokal. In ein Claude Project geladen, mit Lese- und Schreibzugriff.

terminal ~/your-shop 
$ shopify theme pull --live              # raw export
$ claude project init ./theme            # load workspace
$ claude project add ./.claude/workflows
→ workspace ready · 142 files indexed
02 CLAUDE

Sections, Snippets, Templates. File-In, File-Out — kein CLI-Tunnel, keine Black-Box. Du siehst jede Änderung, bevor sie irgendwo landet.

03 PUBLISH

Geänderte Files wandern in ein dupliziertes Theme. Preview, Review, Publish. Live-Theme bleibt unangetastet, bis du es willst.

terminal ~/your-shop 
$ shopify theme push --unpublished       # upload duplicate
→ theme uploaded · id 142978653
$ shopify theme open --preview           # review live
$ shopify theme publish                  # ship it
→ live · 0.84s avg · checkout intact ✓

Verhaltensdaten, die Claude lesen kann.

// faq

Was Founder vor dem Kauf fragen.

Brauche ich Programmierkenntnisse? +
Du brauchst keine, aber du solltest Liquid-Files lesen können. vorflows ist nicht für absolute Beginner — die Zielgruppe sind Founder, die ihren Stack verstehen wollen. Der Skill-Stack im Base-Modul deckt alles ab, was im Workflow gebraucht wird.
Funktioniert das mit jedem Shopify-Theme? +
Ja. Der Workflow operiert auf den Standard-Shopify-Files (sections, snippets, templates, config). Custom-Themes, Premium-Themes, Dawn — alle nutzen dieselbe Struktur. Solange du Theme-Export-Rechte hast, läuft der Prozess.
Was ist, wenn mein Theme schon stark angepasst ist? +
Umso besser für Claude — mehr Kontext, mehr Hebel. Stark angepasste Themes profitieren am meisten, weil App-Replacements und Custom-Sections oft die nächste logische Stufe sind. Der Workflow verändert nichts ohne Review.
Kann ich Module später nachkaufen? +
Ja, aber zum Einzelpreis ohne Bundle-Rabatt. Wer von Anfang an weiß, dass er SEO oder ClarityFlow ohnehin will, fährt mit dem Bundle besser.
Wie lange dauert das Setup? +
Base: ein konzentrierter Nachmittag, bis der erste Theme-Umbau läuft. SEO: ~10 Minuten geführter Setup-Flow. ClarityFlow: ~15 Minuten, sobald die Verhaltensdaten-Quelle steht.
Was passiert, wenn Claude einen Fehler im Theme macht? +
Keine Änderung landet je direkt im Live-Theme. Der Workflow arbeitet immer in einem Duplikat — du reviewst, du publishst. Schiefgehen ohne Review ist im Prozess nicht vorgesehen.
